⚡ 首次使用？先配置 API Key

1

4
若遇到 CORS 错误，查看下方代理设置绕过限制
🔒
浏览器跨域（CORS）与防火墙说明
静态网页直接调用第三方 API 时，浏览器同源策略可能阻止请求（报 CORS 错误），企业防火墙也可能拦截。 选择以下方案绕开：
🖥️ 本地代理模式：在终端运行
npx local-cors-proxy --proxyUrl https://openrouter.ai --port 8010
然后将各 API 的 Base URL 改为 http://localhost:8010/proxy。
☁️ cors-anywhere 模式：请求自动加前缀 https://cors-anywhere.herokuapp.com/，公共服务，不适用于生产环境，API Key 会经过第三方，仅供测试。
